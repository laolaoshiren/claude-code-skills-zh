# 掘金发文风格库（随机轮换，确保每天不重复）

## 风格1: 踩坑日记型
口吻：像在跟同事吐槽，讲自己踩过的坑
语气：随意、真实、带点自嘲
结构：问题 → 踩坑过程 → 发现解决方案 → 总结
禁止：不要说"让我来帮你"，要说"我自己踩过"

## 风格2: 效率提升型
口吻：像在分享自己的生产力工具箱
语气：务实、具体、有数据
结构：之前的痛点 → 发现工具 → 实际效果（省了多少时间）→ 怎么用
禁止：不要说"非常强大"，要给具体数字和对比

## 风格3: 技术深度型
口吻：像在写技术博客，有深度分析
语气：专业但不装，通俗易懂
结构：技术背景 → 原理分析 → 实际应用 → 代码示例
禁止：不要堆砌术语，要让小白也能看懂

## 风格4: 对比评测型
口吻：像在写产品测评，客观对比
语气：中立、有观点但不偏激
结构：场景需求 → 多个方案对比 → 推荐方案 → 使用心得
禁止：不要踩一捧一，客观展示优缺点

## 风格5: 入门教程型
口吻：像在带新人，手把手教
语气：耐心、详细、有耐心
结构：这是什么 → 为什么要学 → 怎么用（步骤）→ 进阶技巧
禁止：不要居高临下，要平等交流

## 风格6: 项目复盘型
口吻：像在做项目总结，分享经验
语气：真实、有反思、有收获
结构：项目背景 → 遇到的问题 → 解决方案 → 复盘收获
禁止：不要吹嘘，要真实反映优缺点

## 风格7: 冷知识/技巧型
口吻：像在分享一个偶然发现的小技巧
语气：轻松、惊喜、"你可能不知道"
结构：发现场景 → 这个技巧 → 实际效果 → 还能这么用
禁止：不要长篇大论，短小精悍

## 写作规则（通用）
1. 开头第一句要抓人，不要"大家好我是XXX"
2. 用第一人称"我"，不要用"我们"
3. 多用具体数字，少用"很多""非常"
4. 代码示例要短，不要贴一屏幕代码
5. 结尾用问题引导评论，不要"感谢阅读"
6. 项目链接自然带出，不要放在最显眼位置
7. 不要使用"首先/其次/最后"这种AI典型结构
8. 段落要短，手机阅读体验好
9. 适当加一些emoji但不要太多
10. 每篇至少有一个真实数据或真实场景
